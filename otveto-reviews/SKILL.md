# Otveto Reviews — анализ отзывов через API Ответо

Скилл ходит в GraphQL API Ответо (`https://api.otveto.ru/graphql`) и достаёт
данные раздела «Анализ отзывов»: магазины продавца, прогоны анализа отзывов и
их результаты (рейтинг, кол-во отзывов, позитив/негатив, темы — плюсы/минусы),
а также ленту отзывов с ответами.

## Подготовка (один раз)
1. Скопируйте `config/.env.example` → `config/.env`.
2. Впишите `OTVETO_TOKEN` — значение заголовка `authorization` из app.otveto.ru
   (DevTools → Network → любой запрос на `/graphql` → Request Headers →
   `authorization`). Токен привязан к аккаунту продавца. См. `references/auth.md`.

`config/.env` в .gitignore — токен в репозиторий не попадёт.

## Скрипты (`scripts/`)
- `stores.sh` — магазины продавца (WB+Ozon): `id` (uuid) и название. С `id`
  работают остальные операции/фильтры.
- `analyses-list.sh [page] [take]` — история анализов. Каждый `items[].id` —
  это **analyticsId** для `analysis-data.sh`. Поля: товар, кол-во отзывов,
  позитив/негатив, дата.
- `analysis-data.sh <analyticsId> [marketplace]` — детали анализа. Сначала
  новый формат (V3: `pluses{name}`, `minuses{name}`, `information{value}` —
  темы плюсов/минусов), при «Аналитике старого формата» автоматически
  откатывается на метрики (рейтинг, кол-во, позитив/негатив, ответы). marketplace
  по умолчанию `WB` (ещё: `OZON`, `YANDEX`, `AVITO`).
- `feedbacks.sh [take] [endCursor]` — лента отзывов с ответами (курсорная
  пагинация): товар, оценка, плюсы/минусы покупателя, текст, дата, ответ и его
  статус. Следующая страница — передать `pageInfo.endCursor`.

Все скрипты печатают JSON ответа (если есть `jq` — красиво; иначе через python3).

## Типовой сценарий «разбери отзывы»
1. `scripts/stores.sh` → понять, какие магазины есть.
2. `scripts/analyses-list.sh 1 20` → найти нужный анализ (по названию товара),
   взять его `id`.
3. `scripts/analysis-data.sh <id> WB` → рейтинг, кол-во, **плюсы/минусы и темы**.
4. (опц.) `scripts/feedbacks.sh 20` → конкретные отзывы и ответы.

Затем агент суммирует: средняя оценка, главные плюсы, главные жалобы, доля
отвеченных, рекомендации.

## Примечания
- API GraphQL; полный справочник операций и полей — `references/operations.md`.
- Анализ отзывов keyed по `analyticsId` (сначала прогон в истории, потом данные).
- Запуск НОВОГО анализа (мутации `CreateWbFeedbackAnalize` / `CreateOzonFeedbackAnalize`)
  и публикация ответов — описаны в `references/operations.md`, но в скрипты не
  вынесены намеренно (это операции записи; добавляйте осознанно).
